Bedrock Inference Profile이란?

Application Inference Profile

  • Bedrock Foundation Model에 대한 프록시 역할
  • 팀/프로젝트/환경별 별도 프로파일 생성
  • CloudWatch ModelId 차원으로 개별 추적
  • 동일 모델, 서로 다른 메트릭 수집

핵심 개념

Team A ──→ Profile A ──┐ ├──→ Claude Sonnet 4.6 Team B ──→ Profile B ──┘ CloudWatch: ModelId = "Profile A ID" → Team A 메트릭 ModelId = "Profile B ID" → Team B 메트릭

Inference Profile 유형

  • Foundation Model ARN 직접 지정
  • 해당 리전에서만 추론 가능
  • 용도: 개발/테스트 환경
# 모델 소스 (단일 리전) arn:aws:bedrock:us-east-1::foundation-model/anthropic.claude-sonnet-4-6
  • System-defined Inference Profile ARN 지정
  • 여러 리전에 자동 라우팅 (처리량/복원력 향상)
  • 용도: 프로덕션 환경 (권장)
# 모델 소스 (크로스 리전) arn:aws:bedrock:us-east-1:ACCOUNT:inference-profile/us.anthropic.claude-sonnet-4-6

왜 이 솔루션이 필요한가?

문제점

  • 여러 팀이 같은 모델 사용 시 사용량 파악 불가
  • 비용 분배가 어려움
  • 성능 이슈의 원인 팀 추적 불가
  • 수동 대시보드 구성은 번거롭고 오류 발생

해결

  • Inference Profile로 팀별 사용량 분리
  • 자동 비용 추정 (Price List API 연동)
  • 실시간 성능 모니터링 (레이턴시, 에러, 스로틀)
  • CDK 배포 한 번으로 전체 자동화

Detail Dashboard

주요 위젯

  • 드롭다운 선택기 — 프로파일별 필터링 (Dashboard Variable)
  • 토큰 사용량 — Input/Output 이중 Y축
  • 비용 추정 — Input/Output/Total (Stacked Area)
  • 핵심 카드 — Invocations, Avg/Min/Max Latency, Errors, Throttles
  • 레이턴시 분포 — Avg, p50, p90, p99
  • 에러 & 스로틀 — Client/Server Errors, Throttles

Dashboard Variable

┌─ Select Profile ──────────┐ │ ▼ deptA_penguin_dev │ │ deptA_penguin_prod │ │ deptB_general │ │ my_team_prod │ └───────────────────────────┘ → 선택한 프로파일의 메트릭만 표시 → $inferenceProfile 변수 활용

Comparison Dashboard

전체 프로파일 비교

  • 컬러 코딩 — 10가지 색상 자동 순환
  • 토큰 소비량 — 전체 프로파일 Input/Output 비교
  • 성능 비교 — Invocation Count, Average Latency
  • 비용 추정 — Period별 비용 + 누적 비용
  • 프로파일별 섹션 — 개별 상세 메트릭

누적 비용 (RUNNING_SUM)

Cumulative Cost (USD) │ ╱── Total │ ╱╱ ── Team A │ ╱╱╱ ── Team B │╱╱╱╱ ── Team C └──────────────── time
RUNNING_SUM() CloudWatch 함수로 시간 경과에 따른 누적 비용 시각화

모니터링 지표 상세

지표설명통계용도
InputTokenCount입력 토큰 수Sum프롬프트 크기 추적
OutputTokenCount출력 토큰 수Sum응답 크기 추적
InvocationsAPI 호출 수SampleCount사용 빈도 파악
모든 지표는 AWS/Bedrock 네임스페이스, ModelId 차원으로 수집됩니다
지표통계설명활용
InvocationLatencyAverage평균 응답 시간전반적 성능 파악
p5050%ile (중앙값)일반적 사용자 경험
p9090%ile대부분의 요청 시간
p9999%ile꼬리 지연 시간 (Tail Latency)
Min / Max최소/최대이상치 탐지
p90과 p99가 중요한 이유: Average만으로는 일부 사용자의 느린 경험을 놓칠 수 있습니다
지표HTTP의미대응
InvocationClientErrors4xx잘못된 요청, 권한 문제요청 형식 / IAM 확인
InvocationServerErrors5xxAWS 내부 에러재시도 로직 구현
InvocationThrottles429요율 제한 초과용량 확보 필요
스로틀이 빈번하다면 → Provisioned Throughput 또는 Cross-Region 프로파일 전환을 고려하세요

비용 추정 메커니즘

비용 계산식

Cost = (InputTokens / 1000 × InputPrice) + (OutputTokens / 1000 × OutputPrice) # 누적 비용 Cumulative = RUNNING_SUM(cost_per_period)
CloudWatch Metric Math를 활용하여 실시간 비용을 계산합니다

가격 조회 우선순위

  1. 수동 오버라이드
    CDK Stack에 설정 (최신 모델용)
  2. AWS Price List API
    ServiceCode=AmazonBedrock 자동 조회
  3. 미적용
    가격 없음 → 비용 위젯 미표시

코드 가이드 - Profile 생성

create_profile.pyimport boto3 client = boto3.client("bedrock", region_name="us-east-1") # 단일 리전 프로파일 생성 response = client.create_inference_profile( inferenceProfileName="deptA_penguin_dev", description="DepartmentA Penguin Project Dev", modelSource={ "copyFrom": "arn:aws:bedrock:us-east-1::foundation-model/anthropic.claude-sonnet-4-6" }, ) print(f"ARN: {response['inferenceProfileArn']}")

CLI 도구도 제공:

bash# 단일 리전 python3 create_profile.py --name my_team_dev # 크로스 리전 (프로덕션 권장) python3 create_profile.py --name my_team_prod --cross-region

코드 가이드 - Profile 호출 (핵심!)

invoke_profile.pyimport boto3, json client = boto3.client("bedrock-runtime", region_name="us-east-1") # ✓ Inference Profile ARN을 modelId로 사용 PROFILE_ARN = "arn:aws:bedrock:us-east-1:ACCOUNT:application-inference-profile/w2m9jvmdyfej" response = client.invoke_model( modelId=PROFILE_ARN, # ← Profile ARN 사용! contentType="application/json", accept="application/json", body=json.dumps({ "anthropic_version": "bedrock-2023-05-31", "max_tokens": 256, "messages": [{"role": "user", "content": "Hello!"}], }), )
modelId에 반드시 Inference Profile ARN을 사용해야 합니다!
Foundation Model ID를 직접 사용하면 팀별 구분이 불가합니다. 스트리밍도 동일: invoke_model_with_response_stream()

코드 가이드 - CDK 배포

가격 자동 조회 메커니즘

조회 우선순위

  1. 수동 오버라이드
    CDK Stack에 설정된 가격 (최신 모델용)
  2. AWS Price List API
    ServiceCode=AmazonBedrock, Feature=On-demand Inference
  3. 미적용
    가격 정보 없음 → 비용 위젯 미표시

최적화

  • 모델명 기반 캐싱
    같은 모델 사용하는 프로파일은 1회만 조회
  • 17개 리전 지원
    US, EU, AP, SA, ME 전체
  • 배포 시마다 갱신
    cdk deploy 실행 시 최신 가격 반영
  • Foundation Model 이름 자동 해석
    GetFoundationModel API → 모델 이름 변환

주의사항 & 제약

  • Dashboard Variable 제약
    필터링/비필터링 위젯 혼합 불가 → Detail/Comparison 분리 이유
  • Price List API 제약
    최신 모델(Claude Sonnet 4, Haiku 4.5 등) 미등록 가능 → 수동 오버라이드 필요
  • 메트릭 지연
    CloudWatch 메트릭은 호출 후 5-10분 후 표시
  • 프로파일 ARN 필수
    기존 모델 ID로 호출하면 프로파일별 추적 불가 — 반드시 변경 필요!
  • 배포 시 갱신
    프로파일 추가/삭제 시 cdk deploy 재실행 필요

Key Takeaways

핵심 요약

  • Inference Profile = 팀별 AI 사용량 분리의 핵심
  • 자동 발견 + 자동 가격 조회 + 자동 대시보드
  • Detail 대시보드 → 프로파일별 심층 분석
  • Comparison 대시보드 → 전체 비교 + 누적 비용
  • CDK 배포 한 번으로 모든 것이 자동화
