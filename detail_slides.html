Why?

Bedrock Application Inference Profile을 운영할 때 겪는 문제

가시성 부족

  • 프로파일별 토큰 사용량 추적 어려움
  • Latency 분포(p50/p90/p99)를 직관적으로 확인할 수 없음
  • 오류/쓰로틀 발생을 실시간 모니터링하기 어려움

비용 관리 어려움

  • 프로파일별 토큰 비용 추정이 수동으로만 가능
  • 누적 비용 추이를 파악하기 어려움
  • Input/Output 토큰 가격이 다른데 구분이 안 됨

운영 부담

  • 프로파일 추가/삭제 때마다 대시보드를 수동 수정
  • 프로파일 ID, 가격 정보를 하드코딩으로 관리

Solution: Automated Dashboard Construct

cdk deploy 한 번으로 프로파일 탐색부터 대시보드 생성까지 자동화

Auto-Discovery: Profile & Pricing

Profile Discovery

Lambda가 배포 시점에 자동 실행:

  • ListInferenceProfiles API 호출
  • typeEquals: APPLICATION 필터
  • status: ACTIVE인 것만 수집
  • Pagination 지원 (100개 이상 가능)

CloudWatch는 프로파일의 short ID를 ModelId 차원으로 사용합니다

Pricing Resolution

1
Manual override 확인 (최우선)
2
GetFoundationModel로 모델명 조회
3
Price List API에서 on-demand 가격 조회
-
미등록 모델은 비용 위젯 비활성

Deployment

Quick Start

# 1. Install
$ npm install

# 2. Configure pricing (optional)
#    lib/...-stack.ts

# 3. Deploy
$ npx cdk deploy

# 4. Cleanup
$ npx cdk destroy

What Happens on Deploy

1
CDK가 Lambda + Custom Resource 생성
2
Lambda가 자동으로 프로파일 탐색
3
Price List API에서 가격 자동 조회
4
CloudWatch에 2개 대시보드 생성
!
프로파일 변경 시 cdk deploy만 재실행

Summary

  • Inference Profile 자동 탐색
  • 토큰 가격 자동 조회
  • 2개 대시보드 자동 생성
  • 토큰 사용량 / 비용 / Latency / 오류 모니터링
  • CDK로 IaC 관리, 스택 삭제 시 자동 정리
